### 3.10 수강 관리 — 서버 연동 전환 (P1 변경)

> P1에서 로컬 스토리지에 임시 저장하던 수강 신청·취소·진도율 정보를 서버 DB에 영구 저장하는 방식으로 전환한다.

---

#### FR-P2-032 수강 신청 (서버 저장)

**설명:** 로그인한 학습자는 강의를 수강 신청할 수 있어야 하며, 수강 정보는 서버에 저장되어야 한다.

**기능 상세 (P1 변경):**
- 학습자(STUDENT)는 강의 상세 페이지에서 "수강 신청" 버튼을 눌러 수강 신청한다.
- 시스템은 수강 정보를 서버 DB(enrollment 테이블)에 저장한다.
- 수강 신청 후 버튼이 "수강 중"으로 변경된다.
- 비로그인 사용자가 수강 신청을 시도하면 로그인 페이지로 이동한다.

**제약 조건:**
- APPROVED 상태의 강의만 수강 신청 가능하다.
- 동일 강의 중복 수강 신청 불가하다.
- 로그인한 사용자만 수강 신청 가능하다.

**예외 처리:**
- 이미 수강 중인 강의 신청 시: `"이미 수강 중인 강의입니다"` (409 Conflict)
- 비로그인 상태 수강 신청 시도 시: `"로그인이 필요합니다"` (401 Unauthorized)

---

#### FR-P2-033 수강 취소 (서버)

**설명:** 학습자는 수강 중인 강의를 취소할 수 있어야 하며, 서버에서 수강 정보가 삭제되어야 한다.

**기능 상세 (P1 변경):**
- 학습자(STUDENT)는 "내 강의" 페이지에서 수강 취소를 요청한다.
- 시스템은 서버 DB에서 해당 수강 정보(enrollment 및 완료 파트 정보)를 삭제한다.
- 취소 전 확인 다이얼로그를 표시한다.

**제약 조건:**
- 본인의 수강 정보만 취소 가능하다.

**예외 처리:**
- 수강 중이지 않은 강의 취소 시도 시: `"수강 중인 강의가 아닙니다"` (404 Not Found)

---

#### FR-P2-034 수강 중인 강의 목록 조회 (서버)

**설명:** 학습자는 서버에 저장된 수강 목록을 조회할 수 있어야 한다.

**기능 상세 (P1 변경):**
- 학습자(STUDENT)는 "내 강의" 페이지에서 자신의 수강 목록을 조회한다.
- 목록에는 강의 제목, 강사 이름, 강의 유형, 태그, 진도율(프로그레스 바)이 표시된다.
- 수강 목록은 수강 신청 날짜 기준 최신순으로 표시된다.

**제약 조건:**
- 로그인한 사용자만 조회 가능하다.

---

#### FR-P2-035 파트 시청 완료 처리 (서버)

**설명:** 학습자가 파트 영상을 끝까지 시청하면 시스템은 서버에 완료 정보를 저장해야 한다.

**기능 상세 (P1 변경):**
- 학습자(STUDENT)가 영상 종료 시점에 도달하면 시스템이 자동으로 해당 파트를 완료 처리한다.
- 완료 처리된 파트 정보는 서버 DB에 저장되어 기기/브라우저가 바뀌어도 유지된다.
- 이미 완료 처리된 파트를 재시청해도 진도율은 변경되지 않는다.

**제약 조건:**
- 해당 강의를 수강 신청한 사용자만 완료 처리 가능하다.
- 로그인한 사용자만 서버 저장 가능하다.

---

#### FR-P2-036 진도율 조회 (서버)

**설명:** 학습자는 서버에 저장된 진도율을 조회할 수 있어야 한다.

**기능 상세 (P1 변경):**
- 진도율 = 완료한 파트 수 / 전체 파트 수 × 100 (%)
- 강의 상세 페이지 및 "내 강의" 페이지에서 서버로부터 진도율을 조회하여 표시한다.
- 강의 시청 화면에서도 현재 진도율을 확인할 수 있다.
