## 5. 비기능 요구사항

### 5.1 성능 요구사항

| 항목 | 기준 |
|------|------|
| 강의 목록 조회 응답 시간 | 2초 이내 |
| 강의 검색 응답 시간 | 2초 이내 |
| 강의 상세 조회 응답 시간 | 2초 이내 |
| 로그인 응답 시간 | 1초 이내 |
| 질문 작성 응답 시간 | 3초 이내 |
| 강의 등록 응답 시간 (썸네일 업로드 포함) | 5초 이내 |
| 파트 시청 완료 처리 응답 시간 | 1초 이내 |

### 5.2 보안 요구사항

- 모든 인증이 필요한 API 요청에는 `Authorization: Bearer <Access Token>` 헤더가 포함되어야 한다.
- 비밀번호는 BCrypt 알고리즘으로 암호화하여 저장하며, 평문으로 저장·전송하지 않는다.
- Access Token의 유효기간은 1시간, Refresh Token의 유효기간은 14일로 제한한다.
- Refresh Token Rotation 방식을 적용하여 토큰 재사용 공격을 방지한다.
- 모든 API 응답에서 역할에 따른 접근 제어(RBAC)를 적용한다.
    - STUDENT: 강의 조회, 수강 관리, 질문 작성/수정/삭제(본인)
    - INSTRUCTOR: STUDENT 권한 + 강의·파트·도안 관리, 답변 관리
    - ADMIN: 전체 관리 권한
- 파일 업로드 시 허용된 확장자(jpg, png, gif) 및 MIME 타입을 검증한다.
- SQL 인젝션, XSS 등 OWASP Top 10 취약점에 대한 방어를 적용한다.
- CORS 설정은 허용된 출처(Origin)에서만 요청을 수락한다.
- HTTPS 통신을 권장한다 (개발 환경 제외).

### 5.3 사용성 요구사항

- 로그인/회원가입 화면은 직관적인 폼 형태로 제공되어야 한다.
- 강사 강의 관리 화면은 파트·도안을 단계적으로 등록할 수 있도록 구성되어야 한다.
- 관리자 페이지는 별도의 레이아웃으로 제공되어야 한다.
- P1에서 구현된 학습자 기능(강의 목록, 영상 시청, 도안 보기 등)의 UX는 변경 없이 유지되어야 한다.

### 5.4 확장성 요구사항

시스템은 P3 단계에서 다음 기능을 추가할 수 있도록 확장 가능하게 설계한다.

- 결제 시스템 (강의 유료화, PG사 연동)
- 이메일 알림 (강사 신청 결과, 강의 승인 결과, 비밀번호 재설정)
- 로깅 및 모니터링
- 배치 처리 (진도율 집계, 통계 등)
- 코드 리팩토링 및 성능 최적화

### 5.5 지원 플랫폼 및 화면 요구사항

- P1과 동일: Android 및 iOS 태블릿/스마트폰 기기 지원
- 관리자 기능은 PC 웹 브라우저 환경을 1차 지원 대상으로 한다.

## 6. 외부 인터페이스 요구사항

### 6.1 YouTube 영상 재생

- P1과 동일하게 유지된다.
- 강의 영상은 YouTube 영상 ID 형태로 DB에 저장되며, 클라이언트는 YouTube Embed URL로 재생한다.

### 6.2 파일 저장소

- 썸네일 이미지 및 질문 이미지는 서버 로컬 파일 시스템 또는 외부 오브젝트 스토리지에 저장한다.
- P1에서 사용한 이미지 저장 방식을 계승하되, 강의 썸네일 업로드 경로를 추가 관리한다.
