### 3.12 관리자 기능

---

#### FR-P2-039 회원 목록 조회

**설명:** 관리자는 가입된 전체 회원 목록을 조회할 수 있어야 한다.

**기능 상세:**
- 관리자(ADMIN)는 전체 회원 목록을 조회한다.
- 목록에는 닉네임, 이메일, 역할, 가입일, 계정 상태(활성/정지)가 표시된다.
- 역할별, 계정 상태별 필터링 및 닉네임·이메일 검색이 가능하다.
- 페이지네이션을 지원한다.

**제약 조건:**
- ADMIN 역할만 접근 가능하다.

---

#### FR-P2-040 회원 상세 조회

**설명:** 관리자는 특정 회원의 상세 정보를 조회할 수 있어야 한다.

**기능 상세:**
- 관리자(ADMIN)는 특정 회원의 닉네임, 이메일, 역할, 가입일, 계정 상태, 등록 강의 수(강사인 경우), 수강 강의 수를 확인한다.

**제약 조건:**
- ADMIN 역할만 접근 가능하다.

---

#### FR-P2-041 강사 신청 승인 / 반려

**설명:** 관리자는 PENDING 상태의 강사 신청을 승인하거나 반려할 수 있어야 한다.

**기능 상세:**
- 관리자(ADMIN)는 강사 신청 목록을 조회하고 각 신청의 자기소개 및 강의 계획을 확인한다.
- 승인 시: 해당 사용자의 역할이 INSTRUCTOR로 변경된다.
- 반려 시: 반려 사유를 입력하며, 해당 사용자의 역할은 변경되지 않는다. 해당 사용자는 재신청이 가능하다.

**제약 조건:**
- ADMIN 역할만 처리 가능하다.
- PENDING 상태의 신청만 처리 가능하다.

**예외 처리:**
- 이미 처리된 신청 재처리 시도 시: `"이미 처리된 신청입니다"` (400 Bad Request)

---

#### FR-P2-043 회원 강제 탈퇴

**설명:** 관리자는 특정 회원 계정을 강제 탈퇴(비활성화) 처리할 수 있어야 한다.

**기능 상세:**
- 관리자(ADMIN)는 회원 계정을 강제 탈퇴(소프트 삭제) 처리한다.
- 강제 탈퇴 시 해당 사용자의 모든 토큰이 무효화된다.
- 강사가 강제 탈퇴되는 경우, 해당 강사의 APPROVED 강의는 HIDDEN 처리된다.
- 탈퇴된 사용자의 질문·답변은 '탈퇴한 사용자'로 표시된다.

**제약 조건:**
- ADMIN 역할만 처리 가능하다.
- ADMIN 계정은 강제 탈퇴 불가하다.

---

#### FR-P2-044 강의 전체 목록 조회 (관리자)

**설명:** 관리자는 상태에 관계없이 전체 강의 목록을 조회할 수 있어야 한다.

**기능 상세:**
- 관리자(ADMIN)는 DRAFT, PENDING, APPROVED, REJECTED, HIDDEN 모든 상태의 강의를 조회한다.
- 목록에는 강의 제목, 강사 이름, 강의 유형, 상태, 수강생 수, 등록일이 표시된다.
- 상태별, 강의 유형별 필터링 및 강의 제목·강사 이름 검색이 가능하다.

**제약 조건:**
- ADMIN 역할만 접근 가능하다.

---

#### FR-P2-045 강의 승인 / 반려

**설명:** 관리자는 PENDING 상태의 강의를 검토하여 승인하거나 반려할 수 있어야 한다.

**기능 상세:**
- 관리자(ADMIN)는 PENDING 상태의 강의 내용(파트 목록, 도안, 설명 등)을 확인한다.
- 승인 시: 강의 상태가 APPROVED로 변경되어 학습자에게 공개된다.
- 반려 시: 반려 사유를 입력하며, 강의 상태가 REJECTED로 변경된다. 강사는 수정 후 재요청 가능하다.

**제약 조건:**
- ADMIN 역할만 처리 가능하다.
- PENDING 상태의 강의만 승인/반려 가능하다.

---

#### FR-P2-046 강의 강제 비공개 처리

**설명:** 관리자는 APPROVED 상태의 강의를 강제로 비공개(HIDDEN) 처리하거나 재공개할 수 있어야 한다.

**기능 상세:**
- 관리자(ADMIN)는 APPROVED 강의를 HIDDEN 처리한다. 비공개 사유를 입력한다.
- HIDDEN 처리된 강의는 학습자 목록에서 즉시 제외된다.
- 기존 수강생의 수강 정보 및 진도율은 유지된다.
- 관리자는 HIDDEN 강의를 APPROVED로 재공개할 수 있다.

**제약 조건:**
- ADMIN 역할만 처리 가능하다.

---

#### FR-P2-047 태그 등록

**설명:** 관리자는 플랫폼에서 사용할 태그를 등록할 수 있어야 한다.

**기능 상세:**
- 관리자(ADMIN)는 새 태그 이름을 입력하여 태그를 등록한다.
- 등록된 태그는 강사가 강의 등록/수정 시 선택할 수 있다.

**제약 조건:**
- 태그 이름: 최대 20자, 중복 불가

**예외 처리:**
- 중복 태그 이름 등록 시: `"이미 존재하는 태그입니다"` (409 Conflict)

---

#### FR-P2-048 태그 수정

**설명:** 관리자는 기존 태그 이름을 수정할 수 있어야 한다.

**기능 상세:**
- 관리자(ADMIN)는 기존 태그 이름을 변경한다.
- 태그 이름 변경 시 해당 태그를 사용 중인 모든 강의에 즉시 반영된다.

**제약 조건:**
- 태그 이름: 최대 20자, 중복 불가

---

#### FR-P2-049 태그 삭제

**설명:** 관리자는 태그를 삭제할 수 있어야 한다.

**기능 상세:**
- 관리자(ADMIN)는 태그를 삭제한다.
- 강의에 연결된 태그를 삭제하면 해당 강의-태그 연결이 자동으로 해제된다.
- 삭제 전 해당 태그를 사용 중인 강의 수를 표시하고 확인 다이얼로그를 노출한다.
