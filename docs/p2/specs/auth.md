## 3.1 인증

---

### FR-P2-001 회원가입

**설명:** 사용자는 이메일과 비밀번호를 이용하여 계정을 생성할 수 있어야 한다.

**기능 상세:**
- 사용자는 이메일, 비밀번호, 닉네임을 입력하여 회원가입을 요청한다.
- 시스템은 입력값 유효성을 검증한 후 계정을 생성한다.
- 회원가입 시 역할은 STUDENT로 자동 지정된다.
- 비밀번호는 BCrypt 알고리즘으로 암호화하여 저장한다.

**제약 조건:**
- 이메일: RFC 5322 형식, 최대 100자, 중복 불가
- 비밀번호: 최소 8자, 영문 + 숫자 조합 필수, 최대 64자
- 닉네임: 최소 2자, 최대 20자, 중복 불가, 특수문자 불가

**예외 처리:**
- 이미 사용 중인 이메일로 가입 시: `"이미 사용 중인 이메일입니다"` (409 Conflict)
- 이미 사용 중인 닉네임으로 가입 시: `"이미 사용 중인 닉네임입니다"` (409 Conflict)
- 비밀번호 형식 오류 시: `"비밀번호는 영문과 숫자를 포함하여 8자 이상이어야 합니다"` (400 Bad Request)
- 필수 항목 누락 시: `"필수 항목을 입력해 주세요"` (400 Bad Request)

### FR-P2-002 로그인

**설명:** 사용자는 이메일과 비밀번호를 이용하여 로그인할 수 있어야 한다.

**기능 상세:**
- 사용자는 이메일과 비밀번호를 입력하여 로그인을 요청한다.
- 시스템은 자격증명을 검증한 후 Access Token과 Refresh Token을 발급한다.
- Access Token 유효기간: 1시간
- Refresh Token 유효기간: 14일
- Refresh Token은 서버 DB에 저장하여 관리한다.
- 클라이언트는 Access Token을 메모리(또는 보안 저장소)에, Refresh Token을 HttpOnly 쿠키 또는 보안 저장소에 보관한다.

**제약 조건:**
- 정지된 계정은 로그인이 불가하다.

**예외 처리:**
- 이메일 또는 비밀번호 불일치 시: `"이메일 또는 비밀번호가 올바르지 않습니다"` (401 Unauthorized)
- 정지된 계정 로그인 시도 시: `"정지된 계정입니다. 관리자에게 문의하세요"` (403 Forbidden)

---

### FR-P2-003 로그아웃

**설명:** 로그인한 사용자는 로그아웃할 수 있어야 한다.

**기능 상세:**
- 사용자가 로그아웃을 요청하면 시스템은 서버에 저장된 해당 Refresh Token을 무효화한다。
- 클라이언트는 보관 중인 Access Token과 Refresh Token을 삭제한다。

**제약 조건:**
- 인증된 사용자만 로그아웃을 요청할 수 있다。

**예외 처리:**
- 유효하지 않은 토큰으로 요청 시: `"유효하지 않은 인증 정보입니다"` (401 Unauthorized)

---

### FR-P2-004 Access Token 갱신

**설명:** 시스템은 Access Token 만료 시 Refresh Token을 이용하여 새로운 Access Token을 발급할 수 있어야 한다。

**기능 상세:**
- 클라이언트는 Access Token 만료 시 Refresh Token을 이용하여 새 Access Token을 요청한다。
- 시스템은 Refresh Token의 유효성과 서버 DB 일치 여부를 검증한 후 새 Access Token을 발급한다。
- Refresh Token도 함께 갱신(Refresh Token Rotation)한다。

**제약 조건:**
- Refresh Token은 1회 사용 후 새 Refresh Token으로 교체된다。

**예외 처리:**
- 만료되었거나 무효화된 Refresh Token 사용 시: `"다시 로그인해 주세요"` (401 Unauthorized)
