# 뜨개질 온라인 강의 플랫폼 기획서 (P1)

**프로젝트명:** LoopLearn  
**버전:** 1.0 (P1)  
**작성일:** 2026-03-12  

---

## 1. 서비스 개요

본 서비스는 뜨개질을 배우고 싶은 사용자가 다양한 강의를 검색하고 학습할 수 있도록 제공하는 **온라인 강의 플랫폼**이다.
사용자는 강의를 검색하고 강의 영상을 시청하면서 실제 뜨개질 작업을 진행할 수 있으며, 학습 중 궁금한 점을 질문으로 남기고 도움을 받을 수 있다.

본 프로젝트의 **P1 단계에서는 학생(사용자) 관점의 기능만 구현**하며, 로그인 기능 없이 미리 데이터베이스에 등록된 강의를 기반으로 서비스가 동작하도록 설계한다.

또한 본 플랫폼은 일반적인 강의 플랫폼과 달리 **뜨개질 학습에 특화된 기능**을 제공하여 사용자가 강의를 보면서 실제 작품을 제작할 수 있도록 돕는 것을 목표로 한다.

---

## 2. 서비스 이름

**LoopLearn**

LoopLearn은 뜨개질의 반복적인 구조인 Loop와 학습을 의미하는 Learn을 결합한 이름으로, 사용자가 반복 학습을 통해 뜨개질 기술을 익히는 플랫폼을 의미한다.

## 3. 개발 환경 및 기술 스택

### Frontend
- Flutter 기반 웹 및 태블릿 앱 애플리케이션
- Flutter의 단일 코드베이스를 활용하여 웹(PC 브라우저)과 태블릿 앱을 동시에 지원한다.
- **앱은 태블릿 기준으로 가로 모드(Landscape)만 지원하며, 세로 모드는 제공하지 않는다.**
- **P1에서는 웹 환경과 태블릿 앱 환경을 함께 개발한다.**

### 지원 플랫폼

| 플랫폼 | 실행 환경 | 화면 방향 | 비고 |
|--------|---------|---------|------|
| 웹 | PC 브라우저 (Chrome 권장) | 제한 없음 | Flutter Web 빌드 |
| 앱 | 태블릿 (Android / iOS) | 가로 모드 전용 | Flutter 앱 빌드, 세로 모드 미지원 |

### Backend
- Spring Boot 기반 REST API 서버
- 강의 정보, 태그, 질문(Q&A) 등의 데이터를 관리하고 프론트엔드에 API 형태로 제공한다.

### Database
- MySQL
- 강의 정보, 강의 파트, 태그, 질문(Q&A), 도안 정보를 관리한다.

## 4. 개발 범위 (P1)

P1 단계에서는 **학생이 사용하는 기본 학습 기능**을 중심으로 구현한다.

### 구현 기능
- 강의 목록 조회
- 강의 검색 (제목, 태그, 강사 이름 기반)
- 강의 상세 조회
- 강의 수강 신청
- 강의 영상 시청
- 수강 중인 강의 목록 조회 (진도율 포함)
- 수강 취소
- 강의 질문(Q&A) 작성 및 조회
- 질문 이미지 첨부 (최대 1장)
- 도안 기반 학습 기능 (단수 카운터, 도안 기호 설명)

### 미구현 기능 (P2/P3 예정)
- 로그인 및 회원가입
- 역할 기반 접근 제어 (학생 / 강사 / 관리자)
- 강의 업로드
- 질문 답변 기능
- 결제 기능

> P1에서는 관리 기능이 없기 때문에 **강의 데이터와 도안 정보는 개발자가 직접 데이터베이스에 입력하여 사용한다.**

## 5. 강의 구조

하나의 강의는 여러 개의 **학습 파트(강의 영상)** 로 구성된다.

예를 들어 인형 뜨기 강의의 경우 다음과 같이 구성될 수 있다.

- 머리 만들기
- 팔 만들기
- 다리 만들기
- 몸통 만들기
- 바느질 및 조립

각 파트는 개별 영상으로 제공되며, 사용자는 필요한 파트를 선택하여 학습할 수 있다.

---

## 6. 강의 유형 (Lecture Type)

본 플랫폼의 강의는 **난이도가 아닌 학습 방식에 따라 세 가지 유형**으로 분류된다.

### 1) 기초 학습 (Stitch Basics)
뜨개질을 처음 시작하는 사용자를 위한 기초 강의이다. 기초적인 뜨개질 기법을 설명하고 학습하는 것을 목표로 한다.

예시 강의
- 코 잡기
- 짧은뜨기 / 긴뜨기
- 겉뜨기
- 늘림 / 줄임 기법

### 2) 작품 제작 (Project Class)
강사의 설명을 따라 하나의 작품을 완성하는 강의 유형이다. 하나의 강의 안에 여러 파트가 존재하며, 각 파트는 작품의 특정 부분을 제작하는 과정으로 구성된다.

예시 강의
- 인형 만들기
- 모자 만들기
- 가방 만들기
- 소품 만들기

### 3) 도안 (Pattern)
영상 강의 없이 **도안만 제공되는 학습 유형**이다. 도안은 텍스트(표/글) 형태로 제공되며, 사용자는 도안을 기반으로 작품을 제작할 수 있다. 제작 과정에서 궁금한 점이 있을 경우 질문(Q&A)을 통해 문의할 수 있다.

예시
- 인형 도안
- 의류 도안
- 가방 도안

## 7. 강의 검색 기능

사용자는 다음과 같은 방식으로 강의를 검색할 수 있다.

| 검색 방식 | 설명 |
|---------|------|
| 강의 제목 검색 | 키워드가 포함된 강의를 조회 |
| 태그 기반 검색 | 단일 태그를 선택하여 해당 태그가 포함된 강의를 조회 |
| 강사 이름 검색 | 강사 이름으로 강의를 조회 |

### 태그 운영 방식
태그는 강사가 강의 등록 시 직접 입력하는 방식으로 운영된다. 태그는 강의의 특징을 나타내며, 검색 시 단일 태그만 선택 가능하다.

태그 예시
- 인형, 옷, 가방, 소품, 장난감, 뜨개기법

## 8. 강의 영상 제공 방식

강의 영상은 **YouTube 영상을 활용하여 제공한다.**

데이터베이스에는 각 강의 파트별로 YouTube 영상 URL이 저장되며, 웹 애플리케이션에서 해당 영상을 불러와 재생한다.

장점
- 별도의 영상 서버 관리 불필요
- 안정적인 영상 스트리밍 제공
- 트래픽 부담 감소

---

## 9. 뜨개질 특화 학습 기능

본 플랫폼은 일반적인 강의 플랫폼과 달리 **뜨개질 학습을 돕는 특화 기능**을 제공한다.

### 1) 강의 시청 화면 구성

강의 시청 화면은 다음과 같은 구조로 구성된다.

```
┌──────────────────────────────┬──────────────────┐
│                              │                  │
│        강의 영상             │   오른쪽 패널    │
│       (YouTube 재생)         │  (강의 목록 or   │
│                              │    도안 보기)    │
│                              │                  │
└──────────────────────────────┴──────────────────┘
```

사용자는 오른쪽 패널에서 두 가지 모드를 전환할 수 있다.
- **강의 목록 보기** : 전체 파트 목록 표시
- **도안 보기** : 강의에 연결된 도안(텍스트/표) 표시

### 2) 강의 목록 보기

사용자가 수강 중인 강의의 **전체 강의 파트 목록**을 확인할 수 있다. 사용자는 원하는 파트를 선택하여 바로 이동할 수 있다.

예시
- 1. 머리 만들기
- 2. 팔 만들기
- 3. 다리 만들기
- 4. 몸통 만들기
- 5. 바느질 및 조립

### 3) 도안 보기

강의 시청 중 오른쪽 패널에서 도안 보기 모드를 선택하면 도안을 확인할 수 있다. 도안은 두 가지 방식으로 제공된다.

**강의 단위 도안**
강의 유형이 "도안(Pattern)"인 경우, 영상 없이 강의 전체에 연결된 도안을 제공한다. 사용자는 오른쪽 패널에서 강의 전체 도안(텍스트/표)을 확인하며 작품을 제작할 수 있다.

**파트 단위 도안 (영상 연동)**
강의 유형이 "기초 학습" 또는 "작품 제작"인 경우, 파트 영상 재생 중 현재 재생 시간에 해당하는 도안이 오른쪽 패널에 자동으로 표시된다. 영상의 특정 구간(start_time ~ end_time)에 맞는 단수의 도안 텍스트가 표시되어, 사용자는 영상을 보면서 동시에 도안을 참고할 수 있다.

도안이 없는 강의의 경우 오른쪽 패널에 도안 모드가 표시되지 않는다.

### 4) 단수 카운터

도안 화면 하단에는 **단수 카운터(Row Counter)** 기능이 제공된다. 사용자가 현재 뜨개질 작업의 진행 단수를 기록할 수 있도록 돕는 기능이다.

```
현재 단수

[-]  12단  [+]
```

- [+] 버튼 : 단수 1 증가
- [-] 버튼 : 단수 1 감소 (0 미만으로 감소하지 않음)
- 초기값 : 0단
- 단수 값은 세션 내에서만 유지된다 (새로고침 시 초기화)

### 5) 도안 기호 설명 기능

도안에는 다양한 뜨개 기호가 사용된다. 사용자가 도안에서 특정 기호를 클릭하면 해당 기호의 의미를 설명하는 팝업이 표시된다.

| 기호 | 설명 |
|------|------|
| sc | 짧은뜨기 |
| inc | 늘림 |
| dec | 줄임 |
| ch | 사슬뜨기 |
| sl st | 빼뜨기 |

## 10. 수강 신청 및 수강 관리 기능

### 수강 신청

사용자는 강의 상세 페이지에서 수강 신청을 할 수 있다.

| 항목 | 설명 |
|------|------|
| 수강 신청 버튼 | 강의 상세 페이지에서 "수강 신청" 버튼을 통해 신청 |
| 중복 신청 방지 | 이미 수강 중인 강의는 "수강 중" 상태로 표시되며 중복 신청 불가 |

> P1 단계에서는 로그인 기능이 없으므로 수강 신청 정보는 브라우저 로컬 스토리지에 임시 저장한다. P2 이후 사용자 인증과 연동하여 서버에 영구 저장한다.

### 수강 중인 강의 목록 조회

사용자는 자신이 수강 신청한 강의 목록을 별도 페이지에서 확인할 수 있다.

- 수강 중인 강의 카드에는 강의 제목, 강사 이름, 강의 유형, 태그, 진도율이 표시된다.
- 진도율은 전체 파트 수 대비 시청 완료한 파트 수를 기준으로 계산한다.

### 수강 취소

사용자는 수강 중인 강의 목록 페이지에서 수강을 취소할 수 있다.

- 수강 취소 시 확인 다이얼로그를 표시한다.
- 취소 확인 후 해당 강의가 수강 목록에서 제거된다.

### 강의 진도율

사용자가 강의를 시청하면 파트 단위로 진도가 기록된다.

- 특정 파트 영상을 끝까지 시청하면 해당 파트가 완료 처리된다.
- 진도율 = 완료한 파트 수 / 전체 파트 수 × 100 (%)
- 진도율은 수강 목록 페이지에서 프로그레스 바 형태로 표시된다.

> P1 단계에서는 진도 정보를 브라우저 로컬 스토리지에 임시 저장한다.

## 11. 질문(Q&A) 기능

사용자는 강의를 학습하면서 궁금한 점을 질문할 수 있다.

### 질문 작성
질문 작성 시 다음 정보를 입력할 수 있다.

| 항목 | 필수 여부 | 제약 조건 |
|------|---------|---------|
| 질문 제목 | 필수 | 최대 100자 |
| 질문 내용 | 필수 | 최대 1000자 |
| 이미지 첨부 | 선택 | 최대 1장, 최대 5MB, jpg/png/gif 허용 |

이미지를 첨부하여 자신의 뜨개질 진행 상황을 공유하거나 문제 상황을 설명할 수 있다.

### 질문 목록 조회
- 강의에 등록된 질문 목록을 최신 등록순으로 표시한다.
- 질문 목록에는 질문 제목과 작성일이 표시된다.

### 질문 상세 조회
- 질문 제목, 내용, 첨부 이미지, 작성일을 확인할 수 있다.

> ※ P1 단계에서는 답변 기능을 구현하지 않는다.

## 12. 데이터 운영 방식 (P1)

P1 단계에서는 관리자 기능을 구현하지 않기 때문에 데이터는 다음과 같은 방식으로 관리된다.

| 데이터 | 관리 방식 |
|--------|---------|
| 강의 정보 | 개발자가 직접 DB에 등록 |
| 강의 파트 및 YouTube URL | 개발자가 직접 DB에 등록 |
| 태그 정보 | 개발자가 직접 DB에 등록 |
| 도안 정보 | 개발자가 직접 DB에 등록 |
| 도안 기호 정보 | 개발자가 직접 DB에 등록 |
| 수강 신청 및 진도 정보 | 브라우저 로컬 스토리지에 임시 저장 (P2에서 서버 연동) |

이를 통해 로그인 및 관리 기능 없이도 플랫폼의 핵심 학습 기능을 테스트할 수 있다.

## 13. 기대 효과

본 플랫폼은 뜨개질 학습에 특화된 기능을 통해 사용자가 단순히 강의를 시청하는 것을 넘어 실제 작품을 제작하면서 학습할 수 있는 환경을 제공한다.

강의 영상, 도안, 단수 카운터, 도안 기호 설명 등의 기능을 함께 제공함으로써 뜨개질 학습의 효율성을 높이고, 초보 사용자도 도안을 쉽게 이해하고 활용할 수 있도록 돕는다.
