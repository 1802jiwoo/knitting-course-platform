# 요구사항 명세서 (SRS)
**프로젝트명:** LoopLearn  
**버전:** 1.0 (P1)  
**작성일:** 2026-03-12  
**작성자:** 개발자  

---

## 1. 개요

### 1.1 프로젝트 이름
LoopLearn

### 1.2 프로젝트 목적
LoopLearn은 뜨개질을 배우고 싶은 사용자가 온라인 강의를 통해 작품 제작 과정을 학습할 수 있도록 지원하는 웹 및 태블릿 앱 기반 강의 플랫폼이다.

사용자는 강의를 시청하며 작품 제작 과정을 학습하고, 도안을 확인하거나 단수를 기록하면서 실제 뜨개질 작업을 진행할 수 있다. 또한 질문 기능을 통해 학습 과정에서 발생하는 궁금증을 해결할 수 있도록 지원한다.

이를 통해 뜨개질을 처음 배우는 사용자부터 다양한 작품 제작을 시도해보고 싶은 사용자까지 체계적인 학습 환경을 제공하는 것을 목표로 한다.

### 1.3 시스템 개요
LoopLearn은 PC 웹 브라우저 및 태블릿 앱에서 실행되는 온라인 강의 플랫폼으로, 사용자는 강의를 검색하고 강의 영상을 시청하며 작품 제작 방법을 학습할 수 있다.

강의는 여러 개의 파트로 구성되며 각 파트에는 강의 영상이 포함된다. 또한 강의에는 도안과 관련된 정보가 제공되어 실제 작품 제작을 도와준다.

본 시스템은 Flutter(모바일 앱) 기반 프론트엔드, Spring Boot 기반 백엔드, MySQL 데이터베이스로 구성된다. 앱은 태블릿 기준 가로 모드 전용으로 제공된다.

### 1.4 범위 (P1)
본 명세서는 P1 단계 기준으로 작성되며, 학습자 관점의 기능만을 포함한다. 로그인, 강의 업로드, 결제, 관리자 기능은 P2/P3에서 구현 예정이다.

### 1.5 용어 정의

| 용어 | 설명 |
|-----|-----|
| 강의 | 하나의 작품 제작 방법을 설명하는 전체 강의 단위 |
| 파트 | 강의를 구성하는 세부 학습 단위 (영상 1개 포함) |
| 도안 | 뜨개질 작업을 위한 패턴 또는 제작 방법 정보 |
| 단수 | 뜨개질에서 진행되는 줄(row)의 수 |
| Q&A | 강의와 관련된 질문 기능 |
| 태그 | 강의의 특징을 나타내는 키워드 (예: 인형, 가방, 뜨개기법) |
| 강의 유형 | 기초 학습 / 작품 제작 / 도안의 세 가지 학습 방식 분류 |

## 2. 전체 시스템 설명

### 2.1 사용자 유형

#### 2.1.1 학습자 (P1 구현 대상)
- 뜨개질에 관심이 있고 배우고 싶은 사용자
- 강의를 시청하고 도안을 활용하여 작품 제작을 진행하는 사용자

#### 2.1.2 강사 (P2 이후 구현 예정)
- 자신의 뜨개질 작품 제작 방법을 강의 형태로 공유하고 싶은 사용자

#### 2.1.3 관리자 (P2 이후 구현 예정)
- 강의 데이터 및 시스템을 관리하는 사용자

```
Flutter App (태블릿 · 가로 모드 전용, 스마트폰 · 세로 모드 전용)
        ↓
   REST API (HTTP/JSON)
        ↓
Spring Boot (Backend)
        ↓
   MySQL Database
```

## 3. 기능 요구사항

### 3.1 강의 목록 조회

**설명:** 사용자는 플랫폼에 등록된 강의 목록을 확인할 수 있어야 한다.

**기능 상세:**
- 시스템은 등록된 강의 목록을 사용자에게 표시한다.
- 각 강의 카드에는 강의 이름, 강사 이름, 강의 유형, 태그 목록이 표시된다.
- 강의 목록은 등록 순서로 표시된다.

**예외 처리:**
- 등록된 강의가 없는 경우 "등록된 강의가 없습니다" 메시지를 표시한다.

---

### 3.2 강의 검색

**설명:** 사용자는 원하는 강의를 검색할 수 있어야 한다.

**검색 방식:**
- 강의 이름 기반 키워드 검색
- 강사 이름 기반 검색

**기능 상세:**
- 키워드 검색은 강의 이름에 해당 키워드가 포함된 강의를 반환한다.
- 검색 조건은 조합하여 사용할 수 있다.

**예외 처리:**
- 검색 결과가 없는 경우 "검색 결과가 없습니다" 메시지를 표시한다.
- 검색어를 입력하지 않고 검색 시 전체 강의 목록을 반환한다.

---

### 3.3 강의 상세 조회

**설명:** 사용자는 강의의 상세 정보를 확인할 수 있어야 한다.

**표시 정보:**
- 강의 제목
- 강사 이름
- 강의 유형 (기초 학습 / 작품 제작 / 도안)
- 강의 설명
- 강의 파트 목록 (파트 번호, 파트 제목)
- 관련 태그 목록

**예외 처리:**
- 존재하지 않는 강의 ID로 접근 시 "강의를 찾을 수 없습니다" 메시지를 표시한다.

---

### 3.4 강의 영상 시청

**설명:** 사용자는 강의 영상을 시청할 수 있어야 한다.

**기능 상세:**
- 각 강의 파트에는 하나의 강의 영상이 포함된다.
- 영상은 YouTube 링크를 통해 재생된다.
- 영상 시청 화면은 중앙에 영상, 오른쪽 패널에 강의 목록 또는 도안을 표시한다.
- 사용자는 오른쪽 패널에서 "강의 목록 보기"와 "도안 보기" 모드를 전환할 수 있다.

**예외 처리:**
- YouTube 영상 로드 실패 시 "영상을 불러올 수 없습니다" 메시지를 표시한다.

---

### 3.5 강의 파트 목록 조회

**설명:** 사용자는 강의를 구성하는 파트 목록을 확인하고 원하는 파트를 선택할 수 있어야 한다.

**기능 상세:**
- 하나의 강의는 여러 개의 파트로 구성된다.
- 파트 목록에는 파트 번호와 파트 제목이 표시된다.
- 사용자는 파트를 선택하여 해당 파트의 영상으로 바로 이동할 수 있다.

---

### 3.6 도안 보기

**설명:** 사용자는 두 가지 유형의 도안을 확인할 수 있어야 한다.

#### 3.6.1 강의 단위 도안
**기능 상세:**
- 강의 유형이 "도안(Pattern)"인 경우, 영상 없이 강의 전체에 연결된 도안을 제공한다.
- 사용자는 오른쪽 패널에서 강의 전체 도안(텍스트/표)을 확인할 수 있다.
- 도안이 없는 경우 "등록된 도안이 없습니다" 메시지를 표시한다.

#### 3.6.2 파트 단위 도안 (영상 연동)
**기능 상세:**
- 강의 유형이 "기초 학습" 또는 "작품 제작"인 경우, 각 파트 영상 재생 중 해당 시간대에 맞는 도안을 오른쪽 패널에 표시한다.
- 도안은 영상의 재생 시간(start_time ~ end_time)에 따라 해당 단수의 도안 텍스트가 자동으로 표시된다.
- 파트 단위 도안은 강사가 강의 등록 시 영상 타임라인에 맞춰 직접 설정하는 형식이므로, 도안이 첨부되지 않은 강의의 경우 오른쪽 패널에 도안이 표시되지 않는다.

---

### 3.7 단수 카운터

**설명:** 사용자는 뜨개질 진행 단수를 기록하고 조작할 수 있어야 한다.

**기능 상세:**
- 도안 보기 화면 하단에 단수 카운터가 표시된다.
- 사용자는 [+] 버튼을 눌러 단수를 1 증가시킬 수 있다.
- 사용자는 [-] 버튼을 눌러 단수를 1 감소시킬 수 있다.
- 단수의 초기값은 0이다.

**제약 조건:**
- 단수는 0 미만으로 감소하지 않는다.
- 단수 카운터 값은 로컬에 저장된다.

---

### 3.8 도안 기호 설명

**설명:** 사용자는 도안에 사용되는 기호의 의미를 확인할 수 있어야 한다.

**기능 상세:**
- 시스템은 뜨개 기호 목록과 각 기호의 설명을 제공한다.
- 사용자가 도안에서 특정 기호를 클릭하면 해당 기호의 설명이 표시된다.

**기호 예시:**

| 기호 | 설명 |
|------|------|
| sc | 짧은뜨기 (short crochet) |
| inc | 늘림 |
| dec | 줄임 |
| ch | 사슬뜨기 |
| sl st | 빼뜨기 |

---

### 3.9 수강 신청

**설명:** 사용자는 강의 상세 페이지에서 해당 강의를 수강 신청할 수 있어야 한다.

**기능 상세:**
- 강의 상세 페이지에 "수강 신청" 버튼이 표시된다.
- 수강 신청 버튼을 누르면 수강 신청 확인 다이얼로그가 표시된다.
- 확인 후 해당 강의가 사용자의 수강 목록에 추가된다.
- 이미 수강 신청한 강의는 버튼이 "수강 중"으로 변경되며 중복 신청이 불가하다.
- 수강 신청된 강의는 "내 강의" 페이지에서 확인할 수 있다.

**제약 조건:**
- P1 단계에서는 로그인 기능이 없으므로 수강 정보는 브라우저 로컬 스토리지에 임시 저장한다.
- 새로고침 및 브라우저 종료 시에도 데이터는 유지된다 (로컬 스토리지 특성).

**예외 처리:**
- 이미 수강 중인 강의에 신청 시 "이미 수강 중인 강의입니다" 메시지를 표시한다.

---

### 3.10 수강 중인 강의 목록 조회

**설명:** 사용자는 수강 신청한 강의 목록을 별도 페이지("내 강의")에서 확인할 수 있어야 한다.

**기능 상세:**
- 내비게이션에서 "내 강의" 메뉴를 통해 수강 목록 페이지에 접근할 수 있다.
- 수강 중인 강의 목록에는 강의 제목, 강사 이름, 강의 유형, 태그, 진도율이 표시된다.
- 진도율은 전체 파트 수 대비 시청 완료한 파트 수를 기준으로 계산하며 프로그레스 바 형태로 표시된다.
- 수강 중인 강의가 없는 경우 "수강 중인 강의가 없습니다" 메시지를 표시한다.

---

### 3.11 수강 취소

**설명:** 사용자는 수강 중인 강의를 취소할 수 있어야 한다.

**기능 상세:**
- 수강 중인 강의 목록 페이지에서 각 강의에 "수강 취소" 버튼이 표시된다.
- 수강 취소 버튼을 누르면 확인 다이얼로그를 표시한다.
- 확인 후 해당 강의가 수강 목록에서 제거되고 진도 정보도 함께 삭제된다.

**예외 처리:**
- 취소 다이얼로그에서 "아니오"를 선택하면 수강 취소가 이루어지지 않는다.

---

### 3.12 강의 진도율 기록

**설명:** 사용자는 강의 시청 중 파트 단위로 진도가 자동 기록되어야 한다.

**기능 상세:**
- 강의 시청 화면에서 특정 파트 영상을 끝까지 시청하면 해당 파트가 완료 처리된다.
- 진도율 = 완료한 파트 수 / 전체 파트 수 × 100 (%)
- 진도율은 수강 목록 페이지의 프로그레스 바로 표시된다.
- 시청 중인 파트는 완료 처리되지 않으며, 영상 종료 시점에 완료 처리된다.

**제약 조건:**
- P1 단계에서는 진도 정보를 브라우저 로컬 스토리지에 임시 저장한다.

---

### 3.13 질문(Q&A)

**설명:** 사용자는 강의와 관련된 질문을 작성하고 목록을 확인할 수 있어야 한다.

#### 3.9.1 질문 작성
**기능 상세:**
- 사용자는 강의 페이지에서 질문을 작성할 수 있다.
- 질문 작성 시 다음 정보를 입력한다.
  - 질문 제목 (필수, 최대 100자)
  - 질문 내용 (필수, 최대 1000자)
  - 이미지 첨부 (선택, 1장, 최대 5MB, jpg/png/gif 허용)

**예외 처리:**
- 제목 또는 내용이 비어 있으면 "필수 항목을 입력해 주세요" 메시지를 표시하고 등록을 막는다.
- 허용되지 않는 파일 형식 첨부 시 "jpg, png, gif 파일만 첨부할 수 있습니다" 메시지를 표시한다.
- 파일 용량 초과 시 "파일 크기는 5MB 이하여야 합니다" 메시지를 표시한다.
**기능 상세:**
- 사용자는 강의에 등록된 질문 목록을 확인할 수 있다.
- 질문 목록에는 질문 제목, 작성일이 표시된다.
- 질문은 최신 등록순으로 정렬된다.

**예외 처리:**
- 등록된 질문이 없는 경우 "등록된 질문이 없습니다" 메시지를 표시한다.

#### 3.9.3 질문 상세 조회
**기능 상세:**
- 사용자는 질문의 상세 내용(제목, 내용, 이미지, 작성일)을 확인할 수 있다.

> ※ P1 단계에서는 답변 기능을 구현하지 않는다.

## 4. 비기능 요구사항

### 4.1 성능 요구사항

| 항목 | 기준 |
|------|------|
| 강의 목록 조회 응답 시간 | 2초 이내 |
| 강의 검색 응답 시간 | 2초 이내 |
| 강의 상세 조회 응답 시간 | 2초 이내 |
| 질문 작성 응답 시간 | 3초 이내 |

### 4.2 사용성 요구사항
- 사용자는 직관적인 인터페이스를 통해 강의를 탐색할 수 있어야 한다.
- 강의 시청 화면에서 영상과 도안을 동시에 확인할 수 있어야 한다.
- 도안 보기와 강의 목록 보기 간 전환이 즉각적으로 이루어져야 한다.
- 앱 환경에서는 태블릿 가로 모드 화면에 최적화된 레이아웃을 제공해야 한다.

### 4.3 확장성 요구사항
시스템은 향후 다음과 같은 기능을 추가할 수 있도록 확장 가능하게 설계한다.

- 사용자 로그인 및 인증 기능 (P2)
- 강의 업로드 기능 (P2)
- 결제 기능 (P3)
- 강의 댓글 및 답변 기능 (P3)

### 4.4 보안 요구사항 (P1 기준)
- P1 단계에서는 로그인 기능이 없으므로 별도의 인증/인가 처리를 하지 않는다.
- 질문 이미지 업로드 시 익명 계정으로 업로드한다.

### 4.5 지원 플랫폼 및 화면 요구사항

#### 앱 (Flutter App · 태블릿)
- Android 및 iOS 태블릿, 스마트폰 기기를 대상으로 한다.

## 6. 데이터 관리 방식 (P1)

P1 단계에서는 관리자 기능이 구현되지 않기 때문에 강의 데이터는 개발자가 직접 데이터베이스(MySQL)에 입력하는 방식으로 관리한다.

등록되는 데이터는 다음과 같다.

| 데이터 | 설명 |
|--------|------|
| 강의 정보 | 강의 제목, 설명, 강사 이름, 강의 유형 |
| 강의 파트 정보 | 파트 제목, 순서, YouTube 영상 URL |
| 태그 정보 | 태그 이름 |
| 도안 데이터 | 도안 제목, 내용 |
| 도안 기호 데이터 | 기호, 설명 |
