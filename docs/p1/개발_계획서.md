# 개발 계획서 (P1)

**프로젝트명:** LoopLearn  
**버전:** 1.0 (P1)  
**작성일:** 2026-03-12  

---

## 1. 프로젝트 개요

본 프로젝트는 뜨개질 학습을 위한 온라인 강의 플랫폼을 개발하는 것을 목표로 한다.
사용자는 플랫폼에서 다양한 뜨개질 강의를 검색하고 강의를 시청하며 실제 작품 제작을 진행할 수 있다.

본 개발 계획서는 **P1 단계 개발 계획**을 중심으로 작성되며, P1에서는 학생 관점의 기능을 중심으로 시스템의 기본 구조와 핵심 기능을 구현한다.

---

## 2. 개발 방식

본 프로젝트는 **개인 프로젝트**로 진행된다.

프로젝트의 전체 설계, 백엔드 개발, 프론트엔드 개발 및 데이터베이스 설계는 모두 단일 개발자가 수행한다.

---

## 3. P1 구현 목표

P1 단계에서는 기획서에서 정의된 **학생 사용자 기능**을 중심으로 구현한다.

### 구현 기능

| 기능 | 설명 |
|------|------|
| 강의 목록 조회 | 등록된 강의 목록을 표시 |
| 강의 검색 | 강의 이름, 강사 이름 기반 검색 |
| 강의 상세 조회 | 강의 제목, 강사, 설명, 파트 목록, 태그 표시 |
| 강의 수강 신청 | 강의 상세 페이지에서 수강 신청 처리 |
| 강의 영상 시청 | YouTube 영상 재생 |
| 강의 파트 목록 조회 | 파트 선택 및 이동 |
| 수강 중인 강의 목록 조회 | 수강 신청한 강의 목록 및 진도율 표시 |
| 수강 취소 | 수강 목록에서 강의 수강 취소 |
| 강의 진도율 기록 | 파트 시청 완료 기준 진도율 계산 및 표시 |
| 도안 보기 | 텍스트(표/글) 형태의 도안 표시 |
| 단수 카운터 | 단수 증가/감소 기록 (로컬 저장) |
| 도안 기호 설명 | 기호 클릭 시 설명 표시 |
| 질문(Q&A) 작성 | 제목, 내용, 이미지 최대 1장 첨부 |
| 질문 목록/상세 조회 | 강의별 질문 목록 및 상세 확인 |

### 미구현 기능 (P2/P3 예정)

- 로그인 및 회원가입
- 역할 기반 접근 제어 (학생 / 강사 / 관리자)
- 강의 업로드 및 관리
- 질문 답변 기능
- 결제 시스템

> P1 단계에서는 로그인 및 관리자 기능을 구현하지 않으며, 강의 데이터는 사전에 데이터베이스에 등록하여 사용한다.

## 4. 개발 환경

### Frontend
- **프레임워크:** Flutter (Web + 태블릿 앱)
- Flutter의 단일 코드베이스를 활용하여 PC 웹 브라우저용 Flutter Web과 태블릿용 Flutter 앱을 동시에 개발한다.
- 앱은 태블릿 기준 **가로 모드(Landscape) 전용**으로 동작하며, `SystemChrome.setPreferredOrientations`를 통해 앱 실행 시 가로 모드를 강제 고정한다.
- 웹은 PC 브라우저 환경(Chrome 권장)에서 실행되며 화면 방향 제한을 두지 않는다.

### Backend
- **프레임워크:** Spring Boot
- Spring Boot를 사용하여 강의 정보, 질문(Q&A), 도안 데이터 등을 관리하는 REST API 서버를 구축한다.

### Database
- **DBMS:** MySQL
- 강의 정보, 태그, 질문(Q&A), 도안 데이터 등을 저장하고 관리한다.

### 영상 제공 방식
- **YouTube 영상 사용**
- 강의 영상은 YouTube 링크를 활용하여 제공하며, 데이터베이스에는 영상 URL을 저장하고 클라이언트에서 해당 영상을 재생한다.

### 개발 도구

| 항목 | 도구 |
|------|------|
| IDE | Android Studio |
| API 테스트 | Postman |
| 버전 관리 | Git (단일 브랜치 - main) |
| 앱 테스트 | Android / iOS 태블릿 에뮬레이터 또는 실기기 |

### 실행 환경

| 플랫폼 | 실행 환경 | 화면 방향 |
|--------|---------|---------|
| 웹 | PC 브라우저 (Chrome 권장) | 제한 없음 |
| 앱 | 태블릿 (Android / iOS) | 가로 모드 전용 |

## 5. 시스템 구조

본 시스템은 **프론트엔드, 백엔드, 데이터베이스로 구성된 3계층 구조**로 설계된다.

```
Flutter Web  (PC 브라우저)
Flutter App  (태블릿 · 가로 모드 전용)
        ↓
REST API (HTTP/JSON)
        ↓
Spring Boot (Backend)
        ↓
MySQL Database
```

### 시스템 동작 방식

1. 사용자는 Flutter Web(PC 브라우저) 또는 Flutter 앱(태블릿 가로 모드)을 통해 서비스를 이용한다.
2. 프론트엔드는 REST API를 통해 백엔드 서버와 통신한다.
3. Spring Boot 서버는 요청을 처리하고 필요한 데이터를 MySQL 데이터베이스에서 조회한다.
4. 조회된 데이터는 JSON 형태로 프론트엔드에 전달된다.

## 6. 데이터베이스 설계

### 테이블 목록

| 테이블명 | 설명 |
|---------|------|
| lecture | 강의 기본 정보 (lecture_type 포함) |
| lecture_part | 강의를 구성하는 파트 (YouTube URL 포함) |
| video | 파트별 YouTube 영상 정보 |
| tag | 태그 정보 |
| lecture_tag | 강의-태그 다대다 관계 테이블 |
| lecture_pattern | 강의 단위 도안 (도안 유형 강의용) |
| part_pattern | 파트 단위 도안 (영상 재생 시간 연동) |
| pattern_symbol | 도안 기호 및 설명 |
| question | 강의별 질문(Q&A) |
| enrollment | 수강 신청 정보 및 진도율 (P1에서는 로컬 스토리지 임시 저장, P2에서 서버 연동) |

### 데이터 입력 방식 (P1)

P1 단계에서는 관리자 기능이 구현되지 않기 때문에 **강의 데이터는 개발자가 직접 데이터베이스에 입력하는 방식**으로 관리한다.

| 도안 기호 설명 | 기호 클릭 시 설명 팝업 표시 여부 |
